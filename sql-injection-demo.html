☠ ATTACK PAYLOADS — try these in the vulnerable form

Classic Bypass

admin' --

Closes the string early, comments out the rest of the query including the password check.

OR-based Bypass

' OR '1'='1

Injects a condition that always evaluates to TRUE, bypassing credential validation entirely.

Tautology + Union

' OR 1=1--

A cleaner tautology injection; works across most SQL databases regardless of whitespace handling.

// HOW IT WORKS

01

THE VULNERABLE QUERY

The server builds the SQL string by directly pasting user input in. An attacker controls part of the query's structure, not just its data.

// ❌ VULNERABLE — never do this
const query = "SELECT * FROM users WHERE username='"
  + req.body.username // ← attacker controls this
  + "' AND password='"
  + req.body.password
  + "'";
db.query(query);

// Input: admin' --
// Result: SELECT * FROM users WHERE username='admin'--' AND password='x'
// ↑ password check completely bypassed!
02

THE INJECTION MECHANISM

The single quote ' terminates the string literal in SQL. -- starts a comment, ignoring everything after. The attacker has rewritten your query.

-- Original intended query:
SELECT * FROM users WHERE username='admin' AND password='secret';

-- Injected query:
SELECT * FROM users WHERE username='admin'--' AND password='x';
-- ↑ everything after -- is a comment, password ignored
03

THE SECURE FIX — PREPARED STATEMENTS

Parameterized queries separate SQL code from data. The database driver handles escaping, so no amount of quotes or special chars can break out of the data slot.

// ✅ SECURE — parameterized query
const query = "SELECT * FROM users WHERE username=? AND password=?";
db.query(query, [req.body.username, req.body.password]);

// Input: admin' --
// Treated as literal string — no injection possible
// Result: no rows found → login denied
